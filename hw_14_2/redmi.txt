Разнесение HW_14.2 по модулям

Я разделил программу на несколько отдельных файлов — модулей
Каждый модуль отвечает за свою часть логики:
human.py — класс Human
student.py — класс Student (наследуется от Human)
group.py — класс Group
main.py — главный файл, в котором выполняются проверки
Такой подход делает проект структурированным и удобным для расширения

2. Класс Human
В нём хранится общая информация о человеке:
имя
фамилия
возраст
пол
И есть метод __str__ который возвращает понятное текстовое описание

3. Класс Student
Наследуется от Human и добавляет:
номер зачётной книжки
Переопределён метод __str__, чтобы корректно выводить студента
Важные добавления по заданию:
✔ Метод сравнения __eq__
Он сравнивает двух студентов по строке, которую возвращает метод __str__
То есть если данные одинаковые — студенты считаются равными
Это нужно для проверки:
#  assert gr.find_student('Jobs') == st1

Метод __hash__
Так как группа хранит студентов внутри set
объекты должны быть "хешируемыми".
Хеш создаётся также на основе строки str(self)
Это решает ошибку типа:
#  TypeError: unhashable type: 'Student'

4. Класс Group
Хранит студентов в set и содержит методы:
добавить студента
найти студента по фамилии
удалить студента
вывести всех студентов (__str__)
Метод удаления использует метод поиска

5. main.py
Главный файл, где я:
создаю студентов
создаю группу
добавляю студентов
печатаю группу
выполняю проверки assert
То есть демонстрирует, что проект работает после разнесения по модулям

Коротко:
Я разнёс проект на модули: Human, Student, Group и main.py.
Добавил студенту методы сравнения __eq__ и __hash__, чтобы объекты можно было сравнивать и хранить в set
В main.py я импортировал классы и проверил работу — поиск, удаление, сравнение и вывод студентов
